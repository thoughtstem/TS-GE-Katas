ts-kata-util

thoughtstem

 (require ts-kata-util) package: ts-kata-util

These are the utilities that help language designers craft a language that is suitable for building a kata collection on top of.

The key is to have lots of examples in the examples directory. Examples are defined with define-example-code. See a language like battle-arena to see how the examples directory can be organized.

syntax

(define-example-code lang example-name expr ...)

Formerly define-racket-code.

lang should be an installable racket package that defines a #lang. Most of the time it will be identical to the package you’re defining examples for. E.g. if you were making examples for battle-arena, you might do something like this:

(define-example-code battle-arena my-example
  (battle-arena-game #:avatar (custom-avatar)))

example-name can be anything, but should be unique across examples.

Defining an example will 1) create a function called run:example-name that will run the example, and 2) write out the code for the example into the directory examples/compiled-example-data/

From there, it can be included into a kata collection with lang->kata-collection.

procedure

(get-example-code lang example-name) → string?

  lang : symbol?
  example-name : symbol?
Returns a code string suitable for passing into, say, pict’s codeblock function. Or scribble’s codeblock function.

procedure

(get-example-names lang) → (listof symbol?)

  lang : symbol?
Returns a list of symbols that correspond to all example names in a collection of examples.

syntax

(define/contract/doc header contract doc body-expr ...)

Basically the same as define/contract except that it also lets you specify documentation which will get automatically placed into the scribble docs for a language.

You’ll need to do some setup before this will work. See langs survival or battle-arena. A few things: Make the language "#lang at-exp racket" so that your docs can be written using @-expressions. In main.rkt provide the extracted documentation. In your manual.scribl file, require the docs and display them.

Once that’s all done, though, you can simply add define/contract/doc wherever you want.

Known limitations: Currently, the define/contract/doc macro only supports the ->i style of contracts.

1 Data Structures

struct

(struct kata (id stimulus response tests)
    #:extra-constructor-name make-kata)
  id : symbol?
  stimulus : stimulus?
  response : response?
  tests : (listof struct?)
A kata is a stimulus and a response, and some (optional) tests for how a student should generate the response after receiving the stimulus.

Although you can construct a kata by hand, building its stimulus, response, tests, etc., it is more customary to use the kata construction tools. See below

struct

(struct response (data)
    #:extra-constructor-name make-response)
  data : any/c
A structure containing data that is intended to be produced as a kata response.

There are currently three subtypes of responses:

  • code

  • say

  • do

struct

(struct stimulus (data)
    #:extra-constructor-name make-stimulus)
  data : any/c
A structure containing data that is intended to be given as a kata stimulus.

There are currently three subtypes of stimuli:

  • hear

  • read

struct

(struct kata-collection (katas)
    #:extra-constructor-name make-kata-collection)
  katas : (listof kata?)
A wrapper around a list of katas.

struct

(struct expression (lang data)
    #:extra-constructor-name make-expression)
  lang : symbol?
  data : any/c
Anything whose data makes sense in some language lang.

Examples:

(expression 'English "How do you code a circle?")

(expression 'racket "(+ 2 2)")

2 Kata Construction Tools

These are tools for building katas of various kinds. Note that many katas begin their life as an example in some language. For that workflow, see the section Lifting Language Examples to Katas.

However, it is also common to create new katas with the tools in this section. This is especially true for unplugged katas, meta katas, teaching katas, and other non-coding katas.

procedure

(make-kata stim resp [tests]) → kata?

  stim : stimulus?
  resp : response?
  tests : (listof any/c) = '()
A low(ish)-level tool for building katas if you already have constructed a stimulus and response.

You can construct stimulus and response structs with various convenience functions, read, say, do. (See below.)

However, it can often be easier to use a function that returns an entire kata, e.g. recite, translate, coach, etc.

Neither way is more correct. Just use the tools that make your job easiest.

procedure

(read data [#:lang lang]) → stimulus?

  data : any/c
  lang : symbol? = 'English
Convenience function for returning a reading stimulus.

Assumed language is English (for now).

procedure

(say d) → response?

  d : any/c
Convenience function for returning a say response.

procedure

(do d) → response?

  d : any/c
Convenience function for returning a do response.

procedure

(set-id id k) → kata?

  id : symbol?
  k : kata?
Sets the kata’s id, returns a new kata.

procedure

(within [#:minutes minutes    
  #:seconds seconds]    
  k) → kata?
  minutes : number? = #f
  seconds : number? = #f
  k : kata?
Adds a timed test to kata k. Supply either #:minutes or #:seconds but not both.

procedure

(recite #:in in #:out out) → kata?

  in : any/c
  out : any/c
Returns a kata with a hear stimulus and a say response.

Respectively the in is used as the data for the stimulus, and out is used as the data for the response.

Example, the core values kata is defined with:

(recite #:in "What are our core values?"
        #:out (list "It's not about my code; it's about what I can code!"
                    "It's not about me; it's about my team."))

procedure

(translate [#:id id]    
  #:in p    
  [#:in-lang from-lang]    
  #:out c    
  [#:out-lang in-lang]) → kata?
  id : symbol? = 'TODO-id
  p : any/c
  from-lang : symbol? = 'English
  c : any/c
  in-lang : symbol? = 'racket
Returns a kata with a read stimulus and a code response.

Respectively the in is used as the data for the stimulus, and out is used as the data for the response. The lang keywords can control the language tag attached to the data.

This function is used for common coding katas:

(translate #:in "How do you calculate 2 plus 2 in Racket?"
           #:out '(+ 2 2))

procedure

(coach k [#:with-materials materials]) → kata?

  k : kata?
  materials : (listof any/c) = '()
Meta alert!

This takes a kata and returns a kata. Useful for creating katas for teachers.

Example:

(coach core-values
       #:with-materials
       '(whiteboard))

procedure

(kata-id->kata-name id) → string?

  id : symbol?
Converts a kata id to a human readable string.

3 Lifting Language Examples to Katas

procedure

(example->kata lang example-id) → kata?

  lang : symbol?
  example-id : symbol?
Returns a kata where the response is the example’s code and the stimulus is unspecified. This is not yet suitable for use in classrooms. The stimulus must be set.

procedure

(lang->kata-collection lang) → kata-collection?

  lang : symbol?
Returns a kata collection where each example in the given lang has been converted to a kata where the response is the example code and the stimulus is unspecified. Basically it maps example->kata over all the examples defined by the langauge.

A typical workflow might use this function, followed by fill-in-stimuli.

procedure

(fill-in-stimuli collection fill-ins ...) → kata-collection?

  collection : kata-collection?
  fill-ins : (or/c symbol? stimulus?)
fill-ins should be an alternating sequence of example ids and stimuli. For each provided id, that kata in collection is given the stimuli that follows the id.

procedure

(has-keyword? kw k) → boolean?

  kw : string?
  k : kata?
Determines if the kata’s stimulus or response, when converted to a string, contains the given keyword. Useful for filtering collections:

(filter-collection (curry has-keyword? "Avatar") my-collection)

procedure

(num-expressions k) → number?

  k : kata?
Returns the number of expressions in the response of the kata (assuming the kata’s response is to code some rackety thing).

Can be useful if you want a rough estimate of difficulty for some kata.

procedure

(find-by-id id collection) → kata?

  id : symbol?

Returns the first matching kata from collection, whose id is id.

procedure

(kata-name k) → string?

  k : kata?
Converts the kata’s id into a human-friendly name by removing dashes and numbers, and by calling string-titlecase.

procedure

(name-contains? k s) → boolean?

  k : kata?
  s : string?
Returns #t if the kata’s name contains s.

syntax

(define-sub-collections
  [collection kata-collection?]
  name
  ...)
Defines and provides an additional kata collection per name.

The name can either be a naked identifier, or an s-expression that starts with an identifier and is followed by a sequence of refinements on the desired sub collection.

Example:

(define-sub-collections battle-arena-katas
  avatar-katas
  enemy-weapon-katas
 
  (enemy-katas
   (not/c (curryr name-contains? "Weapon"))))

syntax

(define/provide header body-expr ...)

Defines something and provides it.

syntax

(define-kata-collection name kata-or-kata-collection ...)

Constructs, defines, and provides a collection named name from the given katas or kata collections.

procedure

(kata->scribble k [#:befores befores]) → scribble?

  k : kata?
  befores : (listof procedure?) = '()
Call this in a scribble document to embed a kata. Use befores to add front matter to katas, e.g. for gamification hints to coaches.

By default the katas are given their own subsection. This can be controlled with befores. See the generated code in rendering.rkt in any kata collection created with raco new-kata-collection COLLECTION-NAME.

procedure

(kata-collection->scribble kc    
  [#:befores befores]) → scribble?
  kc : kata-collection?
  befores : (listof procedure?) = '()
Same as kata->scribble but for a collection.

 (require ts-kata-util/rendering/pict)

procedure

(kata->pict k) → pict?

  k : kata?
Returns a pict representation of the kata.

Can be useful within a scribble doc when talking about katas in a meta way. Sometimes you don’t want the kata to be given its own section. You just want them floating around as picts within a section. (See ts-coach for examples.)
