ts-kata-util

thoughtstem

 (require ts-kata-util) package: ts-kata-util

These are the utilities that help language designers craft a language that is suitable for building a kata collection on top of it.

The main key is to have lots of examples in the examples directory. Examples are defined with define-example-code. See a language like battle-arena to see how the examples directory can be organized.

syntax

(define-example-code lang example-name expr ...)

Formerly define-racket-code.

lang should be an installable racket package that defines a #lang. Most of the time it will be identical to the package you’re defining examples for. E.g. if you were making examples for battle-arena, you might do something like this:

(define-example-code battle-arena my-example
  (battle-arena-game #:avatar (custom-avatar)))

example-name can be anything, but should be unique across examples.

Defining an example will 1) create a function called run:example-name that will run the example, and 2) write out the code for the example into the directory examples/compiled-example-data/

From there, it can be included into a kata collection with lang->kata-collection.

procedure

(get-example-code lang example-name) → string?

  lang : symbol?
  example-name : symbol?
Returns a code string suitable for passing into, say, pict’s codeblock function. Or scribble’s codeblock function.

procedure

(get-example-names lang) → (listof symbol?)

  lang : symbol?
Returns a list of symbols that correspond to all example names in a collection of examples.
